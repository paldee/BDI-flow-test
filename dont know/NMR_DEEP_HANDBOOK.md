## Chapter 1: ฟิสิงส์เชิงเคมีของ 1H NMR และความท้าทายในตัวอย่างชีวภาพ

### 1.1 หลักการพื้นฐานทางกายภาพของ NMR
เมื่อนิวเคลียสของไฮโดรเจน ($^1H$) ซึ่งมีสปินสุทธิ (Net Spin) ไม่เป็นศูนย์ ถูกนำไปวางในสนามแม่เหล็กภายนอกที่มีความเข้มสูง $B_0$ ระดับพลังงานของนิวเคลียสจะแยกออกเป็นสองระดับตามปรากฏการณ์ **Zeeman Effect** ได้แก่ ทิศทางขนานกับสนามแม่เหล็ก (Spin Up, $\alpha$) และทิศทางตรงกันข้าม (Spin Down, $\beta$)

นิวเคลียสจะส่ายรอบแกนสนามแม่เหล็กด้วยความถี่เฉพาะที่เรียกว่า **Larmor Frequency ($\nu_0$)** ซึ่งคำนวณได้จาก:
$$\nu_0 = \frac{\gamma B_0}{2\pi}$$
โดยที่ $\gamma$ คือ Gyromagnetic Ratio ของนิวเคลียสไฮโดรเจน

เมื่อเรากระตุ้นด้วยคลื่นความถี่วิทยุ (RF Pulse) ที่มีความถี่เท่ากับ Larmor Frequency นิวเคลียสจะดูดกลืนพลังงานและเกิดการเปลี่ยนสถานะพลังงาน (Resonance) เมื่อคลื่น RF หยุดลง นิวเคลียสจะผ่อนคลายกลับสู่สถานะเดิม (Relaxation) และแผ่สัญญาณการตอบสนองออกมาในรูปของ **Free Induction Decay (FID)** ซึ่งเป็นสัญญาณตามโดเมนเวลา (Time Domain) สัญญาณ FID นี้จะถูกแปลงด้วยกรรมวิธี **Fourier Transform (FT)** เพื่อให้อยู่ในโดเมนความถี่ (Frequency Domain) เกิดเป็น **NMR Spectrum** ที่ใช้งานในแล็บ

```
               [ 1H Nucleus in magnetic field B_0 ]
                               │
               [ Zeeman splitting of spin states ]
                               │
             [ RF Pulse Excitation @ Larmor Frequency ]
                               │
              [ Relaxation & Free Induction Decay (FID) ]
                               │  (Fourier Transform)
                               ▼
                        [ NMR Spectrum ]
```

### 1.2 Chemical Shift ($\delta$) และมาตรวัด ppm
เนื่องจากอิเล็กตรอนรอบนิวเคลียสจะเคลื่อนที่เพื่อสร้างสนามแม่เหล็กต้านทานสนามภายนอก ทำให้นิวเคลียสแต่ละตำแหน่งในโมเลกุลถูก **บดบัง (Shielding)** จากสนามแม่เหล็กภายนอกไม่เท่ากัน ส่งผลให้ Larmor Frequency แท้จริงของแต่ละนิวเคลียสในโมเลกุลแตกต่างกันไปตามโครงสร้างทางเคมี เราเรียกปรากฏการณ์นี้ว่า **Chemical Shift ($\delta$)**

เพื่อให้อ้างอิงค่าสากลที่ไม่ขึ้นกับความแรงสนามแม่เหล็ก $B_0$ ของเครื่องวัด จึงนิยมแสดงค่าในรูปของส่วนในล้านส่วน (Parts Per Million: ppm):
$$\delta = \frac{\nu_{\text{sample}} - \nu_{\text{ref}}}{\nu_{\text{spectrometer}}} \times 10^6$$
โดยที่ $\nu_{\text{ref}}$ คือความถี่ของสารอ้างอิงมาตรฐาน เช่น Tetramethylsilane (TMS) ซึ่งกำหนดให้มีค่า $\delta = 0\text{ ppm}$

### 1.3 ปัญหาของพีคเลื่อน (Chemical Shift Drift) และสาเหตุทางกายภาพ
Inทางทฤษฎี Chemical Shift ควรอิงตามโครงสร้างทางเคมี 100% แต่ในความเป็นจริง มาตรวัด Larmor Frequency ขึ้นกับระดับความบดบังของอิเล็กตรอนรอบนิวเคลียส ซึ่งมีความไวสูงมากต่อสภาพแวดล้อมทางเคมีกายภาพ (Physicochemical Environment) ของสิ่งส่งตรวจ:
* **pH Variation:** ส่งผลโดยตรงต่อระดับการแตกตัวเป็นไอออน (Protonation/Deprotonation) ของหมู่ฟังก์ชัน เช่น คาร์บอกซิล (Carboxyl, $-\text{COOH} \leftrightarrow -\text{COO}^-$) หรืออะมิโน (Amino, $-\text{NH}_2 \leftrightarrow -\text{NH}_3^+$) การสูญเสียหรือรับโปรตอนทำให้ความหนาแน่นอิเล็กตรอนรอบนิวเคลียสข้างเคียงเปลี่ยนไป ทำให้พีคเกิดการเลื่อนตำแหน่ง
* **Temperature Fluctuation:** อุณหภูมิที่ต่างกันทำให้ความยาวและมุมพันธะมีการสั่นต่างออกไป และมีผลต่อพฤติกรรมการเกิดพันธะไฮโดรเจน (Hydrogen Bonding) กับตัวทำละลาย ซึ่งส่งผลกระทบต่อความบดบังสนามแม่เหล็ก
* **Ionic Strength (Salt Effect):** การปรากฏตัวของไอออนโลหะหนักหรือเกลือเข้มข้นจะดึงดูดกลุ่มเมฆอิเล็กตรอนของสารชีวโมเลกุล นำไปสู่การเลื่อนตำแหน่งของพีคสัญญาณ

ความคลาดเคลื่อนนี้เรียกว่า **Chemical Shift Drift** แม้จะมีค่าเฉลี่ยเพียง $0.01 - 0.05\text{ ppm}$ แต่สำหรับข้อมูลที่มีมิติสูงและพีคที่แคบมาก ความคลาดเคลื่อนเพียงเล็กน้อยนี้สามารถทำให้อัลกอริทึมทั่วไปวิเคราะห์ผิดพลาดอย่างสิ้นเชิง

```
   Clean Standard (pH 7.0) :        ──────/\──────  (e.g., Lactate CH3 @ 1.33 ppm)
   
   Drifted Sample (pH 6.5) :      ──────/\────────  (Shifted left to 1.35 ppm)
```

### 1.4 โครงสร้างพีคตามธรรมชาติ (Lorentzian Peak Shape) และ Spin-Spin Coupling
* **Lorentzian Profile:** สัญญาณ NMR ในโดเมนความถี่ที่เกิดจากการผ่อนคลายแบบเอกซ์โพเนนเชียล (Exponential Decay) ของ FID จะมีรูปร่างทางคณิตศาสตร์ตามสมการ **Lorentzian Function**:
  $$L(x; x_0, A, \gamma) = \frac{A \gamma^2}{(x - x_0)^2 + \gamma^2}$$
  โดยที่ $x_0$ คือตำแหน่งกึ่งกลางพีค (ppm), $A$ คือความแรงสูงสุดของพีค (Amplitude), และ $\gamma$ คือความกว้างครึ่งความสูงของพีค (Half-Width at Half-Maximum: HWHM)
* **Spin-Spin Coupling (J-coupling):** นิวเคลียสข้างเคียงที่มีสปินที่สามารถส่งผลกระทบถึงกันผ่านอิเล็กตรอนในพันธะเคมี จะเหนี่ยวนำให้ระดับพลังงาน Zeeman แยกย่อยออกไปอีก เกิดการแตกตัวของพีคออกเป็นกลุ่มพีคย่อย (Multiplet) เช่น Singlet, Doublet, Triplet, Quartet ตามกฎ $N+1$ ของสปินเพื่อนบ้าน อัตราส่วนความสูงของพีคย่อยในกลุ่มจะสอดคล้องกับ **สามเหลี่ยมปาสกาล (Pascal's Triangle)** เช่น Doublet (1:1), Triplet (1:2:1), Quartet (1:3:3:1) เสมอ

### 1.5 ปรากฏการณ์ Ghost Peaks
**Ghost Peaks** คือ พีคที่ไม่ได้เกิดจากสารเคมีเป้าหมายในสิ่งส่งตรวจทางชีวภาพ แต่เกิดขึ้นจาก:
* สิ่งปนเปื้อนในตัวอย่าง (Contaminants) เช่น สารเคมีล้างหลอดแก้ว หรือตัวทำละลายเกรดต่ำ
* ความบกพร่องของอุปกรณ์สเปกโทรมิเตอร์ เช่น สัญญาณแทรกซ้อน (Spurious signals) หรือความเข้มสนามแม่เหล็กไม่สม่ำเสมอ (Shimming errors)
* ปัญหาการประมวลผลสัญญาณผิดพลาด เช่น การกำหนดมุมเฟสไม่ถูกต้อง (Phase correction error) ทำให้เกิดพีคกลับหัวหรือพีคแปลกปลอม

ในแง่ฟิสิกส์เคมี **Ghost Peak มักจะขัดแย้งกับกฎธรรมชาติอย่างชัดเจน** เช่น ปรากฏตัวเดี่ยวๆ โดยไม่มีสัญญาณของโครงสร้าง J-coupling ที่สอดคล้อง หรือมีสัดส่วนอัตราส่วนความสูง (Coupling ratio) ที่เป็นไปไม่ได้ หรือมีความกว้างพีคแคบเกินขอบเขตทางกายภาพของสเปกโทรมิเตอร์

## Chapter 2: การจัดการเป้าหมายวิเคราะห์ของโจทย์และการก้าวผ่านข้อจำกัดทางกายภาพ

### 2.1 ลักษณะข้อมูลตัวอย่างและโครงสร้างข้อมูลสถิติมิติมหาศาล ( \\ll P$)
ชุดข้อมูลสัญญาณ NMR (Nuclear Magnetic Resonance) จากสถาบันฟีโนมแห่งชาติจะแสดงตัวอย่างใหใฐ้เห็นภาพในรูปของไฟล์ PDF ชื่อ Domain_1_processed_NMR_spectrum.pdf เพื่อใหใฐ้เข้าใจหน้าตาของสัญญาณสเปกตรัมเชิงทัศนูปกรณ์ ส่วนตัวข้อมูลสำหรับใช้งานจริงจะเป็นเมทริกซ์ตัวเลข 2 มิติ (Samples × Signals) ที่มีมิติความละเอียตสัญญาณสูงกว่า 20,000+ จุดพิกัดต่อหนึ่งตัวอย่าง
* **รูปแบบข้อมูลใช้งานจริง (Operational Data Format):** ข้อมูล NMR Pattern สำหรับฝึกสอนและทำนายจริงจะถี่จัดเตรียมและจัดส่งมาในรูปแบบตารางตัวเลข 2 มิติ เช่น ไฟล์ CSV, Parquet หรืออาเรย์ NumPy ความละเอียตสูงตรงสู่ระบบวิเคราะห์ โดยไม่จำเป็นต้องใช้สคริปต์สกัดข้อมูลจากไฟล์ PDF ทัศนูปกรตัวนีๅบเชิงภาพนี้ช่วยใหใฐ้เราเห็นตำแหน่งเคมีฟิสิกส์เป้าหมายเพื่อสร้าง Sim2Real Simulator ไดใ้อย่างถูกต้อง
* **คำสาปแห่งมิติ ( \\ll P$):** ตารางข้อมูล NMR Pattern มีโครงสร้างเป็นเมทริกซ์ 2 มิติ (Samples × Signals) ที่มีมิติความกว้าฟีเจอร์มากกว่า **20,000++ คอลัมน์ต่อหนึ่งตัวอย่าง** ส่งผลให้เกิดปัญหาสถิติโอเวอร์ฟิต และเกิดความสัมพันธ์เชิงเส้นสูงกันเอง (Multicollinearity) ระหว่างสัญญาณที่อยู่ใกล้เคียง### 2.2 การประมวลผลตอบสนองเป้าหมายวิเคราะห์ 5 ประการของโครงการ (The 5 Analytical Tasks)
เพื่อตอบรับเงื่อนไขการประเมินนวัตกรรมและความสมบูรณ์ของระบบ แพลตฟอร์มนี้ได้จัดวางโครงสร้างคณิตศาสตร์เพื่อคลี่คลายโจทย์ทั้ง 5 ประการอย่างมีประสิทธิภาพ:

1. **Feature Selection (การเลือกแกนพิกัดสัญญาณที่สำคัญ):**
   * *หลักการ:* ใช้ `SequenceAwareEncoder` ร่วมกับ Convolution 1 มิติ และ `AdaptiveAvgPool1d` บีบอัดมิติข้อมูลสัญญาณดิบจาก 20,000+ จุด เหลือเพียงพื้นที่แฝง (Latent Space) ขนาด 512 มิติ เพื่อคัดแยกนอยส์และตัดเบสไลน์ที่ไม่มีนัยสำคัญทางชีวภาพออกไป
2. **Pattern Recognition / ML (การจัดระเบียบสัญญาณเลื่อนหลบ):**
   * *หลักการ:* ใช้แบบจำลองระบบพลศาสตร์ความชันต่อเนื่อง **Neural ODE Solver (LatentSpaceODESolver)** ในพื้นที่แฝง 512 มิติ เพื่อเรียนรู้การไหลเคลื่อนตัวของตำแหน่งยอดคลื่นสัญญาณที่เลื่อนหลุด (Chemical Shift Drift) ที่เกิดจาก pH หรืออุณหภูมิ และดึงค่ากลับคืนแกนอ้างอิงมาตรฐานโดยอัตโนมัติ
3. **Compound Classification (การจำแนกรูปแบบสารแม่นยำ):**
   * *หลักการ:* ใช้ `SpectrumDecoder` แปลงสัญญาณกลับสู่พิกัด ppm และป้อนข้อมูลเข้าโมเดลประเมินความสมเหตุสมผลเชิงฟิสิกส์เคมี **LocalizedPatchEBM (Energy-Based Model)** เพื่อแยกแยะและ suppress สัญญาณปนเปื้อน (Ghost peaks) 
4. **Biomarker Discovery (การค้นพบและระบุตัวบ่งชี้โรค):**
   * *หลักการ:* นำสเปกตรัมที่ผ่านการจัดเรียงและคัดแยกสิ่งรบกวนด้วย EBM ไปหาค่าอินทิกรัลเทียบพื้นที่ยอดพีค เพื่อชี้ตัวสารบ่งชี้โรคที่สำคัญผ่าน **คะแนนความเชื่อมั่นผสมแบบไฮบริด (Hybrid Score)** ร่วมกับ **Constrained Dynamic Time Warping (DTW)** และ Peak Bipartite Assignment
5. **Workflow Development (ระบบวิเคราะห์บูรณาการระดับทางคลินิก):**
   * *หลักการ:* บูรณาการระบบประมวลผลทั้งหมดให้โต้ตอบได้ผ่าน **Astro Web Dashboard (Clinical Light-Theme Workstation)** และส่งออกผลเป็น **Structured JSON Diagnostic Payload (clinical_report.json)** ระดับ TRL 4/5 สำหรับอัปโหลดเข้าสู่ EHR โรงพยาบาลจริง พร้อมวงจรส่งค่า Telemetry อัตโนมัติ

### 2.3 โครงสร้างระบบจำลอง Sim2Real ในแบบแผน TRL 4/5
เพื่อแก้ปัญหาการขาดแคลนข้อมูลทางคลินิกที่มีฉลากกำกับอย่างแม่นยำ แพลตฟอร์มนี้จึงนำหลักการ **Sim2Real (Simulation to Reality)** มาใช้ โดยสร้างตัวจำลองเชิงฟิสิกส์เพื่อสร้างข้อมูลสเปกตรัม NMR สังเคราะห์ที่มีการจำลองข้อจำกัดจากเครื่องวัดและสิ่งรบกวนทางคลินิกของจริง

โมดูล `NMRDataSimulator` ดำเนินการสร้างข้อมูลสเปกตรัม 1D $^1H$ NMR ความยาว $4,000$ จุด (ลงความละเอียดเพื่อความเร็วบน POC แต่อัลกอริทึมรองรับ $20,000$ จุด) ผ่าน 3 กลไกย่อย:

1. **การคำนวณสัญญาณชีวภาพบริสุทธิ์ (Lorentzian Profiling):**
   การระบุคลังข้อมูลสารบ่งชี้โรคทางคลินิก (Clinical Biomarkers) สากล อิงตามฐานข้อมูล HMDB เช่น Sucrose, Glucose, Valine, Alanine สารแต่ละตัวจะถูกประกอบขึ้นจากลักษณะพีคย่อยทางฟิสิกส์จริงตามตำแหน่ง ความกว้าง และอัตราส่วนความสูง:
   $$clean\_spectrum(x) = \sum_{c \in \text{compounds}} w_c \sum_{p \in \text{peaks}_c} h_{p} \cdot \frac{\gamma_{p}^2}{(x - x_{p})^2 + \gamma_{p}^2}$$
   โดยที่ $w_c$ คือปริมาณความเข้มข้นสัมพัทธ์ของสารในกลุ่มสิ่งส่งตรวจนั้นๆ

2. **การฉีดความคลาดเคลื่อนทางกายภาพแบบสุ่ม (Stochastic Physical Disturbances):**
   * **Stochastic Shift Drift:** สุ่มค่าการเลื่อนของพีคสัญญาณ $\Delta_{\text{drift}} \sim U(-\text{max\_drift}, \text{max\_drift})$ แล้วทำการเลื่อนสเปกตรัมผ่านการประมวลผลอาร์เรย์ (Array shift) เพื่อสร้างความท้าทายในการจัดตำแหน่ง
   * **Ghost Peak Injection:** ทำการฉีดสิ่งรบกวนที่มีความกว้างพีคแคบมากและความเข้มสูง ณ ตำแหน่งที่แปลกแยก เช่น ในเขตคาร์โบไฮเดรต ($4.15\text{ ppm}$) เพื่อใช้สำหรับทดสอบโมเดลตรวจสอบกฎฟิสิกส์ (Stage 3 EBM)
   * **Additive Baseline Noise:** เพิ่มสัญญาณรบกวนประเภท White Gaussian Noise $N(0, \sigma^2)$ เพื่อแทนสัญญาณรบกวนจากสัญญาณคลื่นแม่เหล็กไฟฟ้าแทรกสอดภายนอก

```python
# คัดย่อจาก generate_synthetic_nmr.py เพื่อแสดงสูตรการสร้างสัญญาณเชิงกายภาพ
def generate_lorentzian_peak(self, x0, intensity, lw):
    """คำนวณรูปร่างพีคทางฟิสิกส์ตาม Lorentzian Function"""
    return intensity * (lw**2 / ((self.ppm_axis - x0)**2 + lw**2))
```

* **pH Variation:** ส่งผลโดยตรงต่อระดับการแตกตัวเป็นไอออน (Protonation/Deprotonation) ของหมู่ฟังก์ชัน เช่น คาร์บอกซิล (Carboxyl, $-\text{COOH} \leftrightarrow -\text{COO}^-$) หรืออะมิโน (Amino, $-\text{NH}_2 \leftrightarrow -\text{NH}_3^+$) การสูญเสียหรือรับโปรตอนทำให้ความหนาแน่นอิเล็กตรอนรอบนิวเคลียสข้างเคียงเปลี่ยนไป ทำให้พีคเกิดการเลื่อนตำแหน่ง
* **Temperature Fluctuation:** อุณหภูมิที่ต่างกันทำให้ความยาวและมุมพันธะมีการสั่นต่างออกไป และมีผลต่อพฤติกรรมการเกิดพันธะไฮโดรเจน (Hydrogen Bonding) กับตัวทำละลาย ซึ่งส่งผลกระทบต่อความบดบังสนามแม่เหล็ก
* **Ionic Strength (Salt Effect):** การปรากฏตัวของไอออนโลหะหนักหรือเกลือเข้มข้นจะดึงดูดกลุ่มเมฆอิเล็กตรอนของสารชีวโมเลกุล นำไปสู่การเลื่อนตำแหน่งของพีคสัญญาณ

ความคลาดเคลื่อนนี้เรียกว่า **Chemical Shift Drift** แม้จะมีค่าเฉลี่ยเพียง $0.01 - 0.05\text{ ppm}$ แต่สำหรับข้อมูลที่มีมิติสูงและพีคที่แคบมาก ความคลาดเคลื่อนเพียงเล็กน้อยนี้สามารถทำให้อัลกอริทึมทั่วไปวิเคราะห์ผิดพลาดอย่างสิ้นเชิง

```

```

### 1.4 โครงสร้างพีคตามธรรมชาติ (Lorentzian Peak Shape) และ Spin-Spin Coupling
* **Lorentzian Profile:** สัญญาณ NMR ในโดเมนความถี่ที่เกิดจากการผ่อนคลายแบบเอกซ์โพเนนเชียล (Exponential Decay) ของ FID จะมีรูปร่างทางคณิตศาสตร์ตามสมการ **Lorentzian Function**:
  $$L(x; x_0, A, \gamma) = \frac{A \gamma^2}{(x - x_0)^2 + \gamma^2}$$
  โดยที่ $x_0$ คือตำแหน่งกึ่งกลางพีค (ppm), $A$ คือความแรงสูงสุดของพีค (Amplitude), และ $\gamma$ คือความกว้างครึ่งความสูงของพีค (Half-Width at Half-Maximum: HWHM)
* **Spin-Spin Coupling (J-coupling):** นิวเคลียสข้างเคียงที่มีสปินที่สามารถส่งผลกระทบถึงกันผ่านอิเล็กตรอนในพันธะเคมี จะเหนี่ยวนำให้ระดับพลังงาน Zeeman แยกย่อยออกไปอีก เกิดการแตกตัวของพีคออกเป็นกลุ่มพีคย่อย (Multiplet) เช่น Singlet, Doublet, Triplet, Quartet ตามกฎ $N+1$ ของสปินเพื่อนบ้าน อัตราส่วนความสูงของพีคย่อยในกลุ่มจะสอดคล้องกับ **สามเหลี่ยมปาสกาล (Pascal's Triangle)** เช่น Doublet (1:1), Triplet (1:2:1), Quartet (1:3:3:1) เสมอ

### 1.5 ปรากฏการณ์ Ghost Peaks
**Ghost Peaks** คือ พีคที่ไม่ได้เกิดจากสารเคมีเป้าหมายในสิ่งส่งตรวจทางชีวภาพ แต่เกิดขึ้นจาก:
* สิ่งปนเปื้อนในตัวอย่าง (Contaminants) เช่น สารเคมีล้างหลอดแก้ว หรือตัวทำละลายเกรดต่ำ
* ความบกพร่องของอุปกรณ์สเปกโทรมิเตอร์ เช่น สัญญาณแทรกซ้อน (Spurious signals) หรือความเข้มสนามแม่เหล็กไม่สม่ำเสมอ (Shimming errors)
* ปัญหาการประมวลผลสัญญาณผิดพลาด เช่น การกำหนดมุมเฟสไม่ถูกต้อง (Phase correction error) ทำให้เกิดพีคกลับหัวหรือพีคแปลกปลอม

ในแง่ฟิสิกส์เคมี **Ghost Peak มักจะขัดแย้งกับกฎธรรมชาติอย่างชัดเจน** เช่น ปรากฏตัวเดี่ยวๆ โดยไม่มีสัญญาณของโครงสร้าง J-coupling ที่สอดคล้อง หรือมีสัดส่วนอัตราส่วนความสูง (Coupling ratio) ที่เป็นไปไม่ได้ หรือมีความกว้างพีคแคบเกินขอบเขตทางกายภาพของสเปกโทรมิเตอร์

## Chapter 3: Stage 1 — Sequence-Aware 1D-CNN Encoder (การสกัดรอยพิมพ์สเปกตรัมที่รักษาความสัมพันธ์เชิงพื้นที่)

### 3.1 ข้อจำกัดของโครงสร้างเดิม (MLP & Dense layers)
ในสถาปัตยกรรมแบบแรกเริ่ม (Baseline) มักนำข้อมูลสัญญาณ 1D ขนาดใหญ่มารีดแบน (Flatten) ทันที แล้วประมวลผลด้วย Linear/Dense Layer วิธีการนี้ละเลยข้อเท็จจริงที่ว่าสัญญาณ NMR เป็นข้อมูลประเภทอนุกรมเวลาที่มีความสัมพันธ์เชิงตำแหน่ง (Spatial/Spatial-temporal Sequence) บนแกน ppm:

                    │

                    │

                    │
            Conv1D (Kernel 7, Stride 2)   --> สกัดฟีเจอร์ระดับกลางเฉพาะบริเวณ
                    │
            GroupNorm (16 groups)
                    │
                  GELU
                    │
            AdaptiveAvgPool1D (128)      --> บีบอัดขนาดสเปกตรัมเชิงลึกให้อยู่ในขนาดฟิกซ์
                    │
            Linear (64*128 -> 512)       --> แปลงเป็นเวกเตอร์แฝง (Latent Embedding)
                    ▼
          [ Latent Features: Batch x 512 ]
```

* **ทำไมต้อง Group Normalization?**
  ในการประมวลผลสเปกตรัม Batch Size มักมีขนาดเล็กเนื่องจากข้อจำกัดปริมาณตัวอย่างแล็บ การใช้ Batch Normalization จะทำให้สถิติตัวแปรผันผวนรุนแรง โมเดลจึงใช้ `GroupNorm` ซึ่งแบ่งแชนเนลออกเป็นกลุ่มย่อยเพื่อหาค่าเฉลี่ยและส่วนเบี่ยงเบนมาตรฐาน ช่วยให้การฝึกฝนเสถียร 100% โดยไม่ขึ้นกับขนาด Batch
* **ทำไมต้อง GELU (Gaussian Error Linear Unit)?**
  GELU ($\text{GELU}(x) = x \cdot \Phi(x)$) ให้ความเรียบและพฤติกรรมที่เป็นไปตามธรรมชาติมากกว่า ReLU โดยไม่ส่งผลให้เกิดปัญหาขั้วประสาทตาย (Dying ReLU) เมื่อประมวลผลบริเวณสัญญาณราบเรียบที่ไม่มีพีค (Zero baseline values)

### 3.3 กฎห้ามใช้ Skip Connections จาก Encoder สู่ Decoder ในงานจัดแนว (No-Shortcut Rule)
ในโมเดล Autoencoder ทั่วไป (เช่น U-Net) มักมีขั้วข้าม (Skip Connections) จากเลเยอร์แรกของ Encoder ไปยังเลเยอร์สุดท้ายของ Decoder เพื่อช่วยส่งข้อมูลความละเอียดสูงกลับไปช่วยลดความเบลอของภาพ

> [!CAUTION]
> **สำหรับงาน NMR Spectrum Peak Alignment ห้ามใช้ Skip Connections โดยเด็ดขาด!**
> หากมี Skip Connections สัญญาณที่ถูกรบกวนหรือขยับตำแหน่ง (Shifted peaks) จะไหลผ่านเส้นทางลัดข้ามผ่าน Neural ODE Solver ไปโผล่ที่ผลลัพธ์ปลายทางทันที ทำให้โมเดลล้มเหลวในการจัดตำแหน่ง เนื่องจากเกิดการรั่วไหลของข้อมูลตำแหน่งคลาดเคลื่อนเดิม (Spatial leakage) บังคับให้ข้อมูลสเปกตรัมทั้งหมดต้องถูกส่งผ่านเวกเตอร์แฝง 512 มิติ และไหลผ่านวิถีความต่อเนื่องของ Neural ODE ก่อนเท่านั้น

## Chapter 4: Stage 2 — Continuous Alignment via Neural ODEs (การจัดแนวแกนสเปกตรัมบนพิกัดต่อเนื่อง)

### 4.1 ทฤษฎีเบื้องหลัง Neural Ordinary Differential Equations (Neural ODEs)
นำเสนอครั้งแรกโดย Chen et al. (2018) แนวคิดของ Neural ODE คือการเปลี่ยนสถาปัตยกรรมการเรียนรู้ของโครงข่ายประสาทเทียมแบบชั้นแยกส่วน (Discrete Layers เช่น ResNet) ให้กลายเป็นระบบการคำนวณที่ต่อเนื่อง (Continuous-Depth Model)

ในโครงข่ายประสาทเทียมปกติ:
$$h_{t+1} = h_t + f(h_t, \theta_t)$$
เมื่อเราลดระยะห่างระหว่างสเต็ปการส่งต่อข้อมูลลงให้มีขนาดเล็กมากจนเข้าใกล้ศูนย์ ($\Delta t \to 0$) สมการความสัมพันธ์จะเปลี่ยนรูปเป็นสมการเชิงอนุพันธ์อันดับหนึ่ง:
$$\frac{dh(t)}{dt} = f_{\theta}(h(t), t)$$
โดยที่ $f_{\theta}$ คือโครงข่ายประสาทเทียมที่เรียนรู้ค่าความชันสนามเวกเตอร์ (Vector field gradient) บนสเต็ปเวลาต่อเนื่องใดๆ จาก $t = 0 \to T$

การหาค่าผลลัพธ์สุดท้าย $h(T)$ จะเปลี่ยนจากการคำนวณผ่านชั้นโมเดลทีละชั้น เป็นการใช้อัลกอริทึมแก้สมการเชิงอนุพันธ์ (ODE Solver) คำนวณหาปริพันธ์ต่อเนื่อง:
$$h(T) = h(0) + \int_{0}^{T} f_{\theta}(h(t), t) dt$$

```
   Discrete ResNet:       h_0 ────► [ Layer 1 ] ────► h_1 ────► [ Layer 2 ] ────► h_2
   
   Continuous Neural ODE: h(0) ───[ Continuous ODE Integration Trajectory ]───► h(T)
```

### 4.2 กลไกการจัดแนวด้วยฟังก์ชันพลศาสตร์ LatentSpaceODESolver
ในแพลตฟอร์มนี้ `LatentSpaceODESolver` ถูกขับเคลื่อนโดยโครงข่ายขนาดเล็กที่ประมาณทิศทางความชันเพื่อแก้พีคเบี่ยงเบน (Peak shift correction) ในพื้นที่ Latent Space ขนาด 512 มิติ:
* **ความเข้ากันทางฟิสิกส์:** ตัวคำนวณประมาณให้วิถีทางเดินเสมือนจำลองแรงดันทางฟิสิกส์ที่บีบให้พีคที่ขยับตำแหน่งเคลื่อนตัวกลับไปยังจุดสมดุลเดิม (Ground truth alignment coordinate)
* **สูตร Euler Integration 4 สเต็ป:** เพื่อรับประกันสิทธิภาพความเร็วในการประมวลผลและการใช้หน่วยความจำต่ำสุด POC กำหนดโครงสร้าง Euler Integration เป็นขอบเขตการคำนวณ:
  $$h_{k+1} = h_k + \Delta t \cdot f_{\theta}(h_k)$$
  โดยที่ $\Delta t = 0.1$ และสเต็ปสูงสุดควบคุมไม่เกิน 4 ขั้น

การทำเช่นนี้เปรียบเหมือนการปล่อยให้เวกเตอร์แฝงไหลผ่านสนามแรงโน้มถ่วงแบบต่อเนื่องจนเกิดการตกตะกอนในสภาวะที่ไม่มีความคลาดเคลื่อนทางกายภาพ

## Chapter 5: Stage 3 — Physics-Constrained Spectrum Reconstruction & Energy-Based Models (EBM)

### 5.1 การคืนมิติกราฟสเปกตรัมด้วย SpectrumDecoder
หลังจากเวกเตอร์แฝงถูกแก้ไขตำแหน่งความคลาดเคลื่อนทางฟิสิกส์แล้ว เวกเตอร์ $h(T)$ จะต้องถูกแปลงกลับสู่มิติภายนอกผ่าน **SpectrumDecoder**:
1. ขยายเวกเตอร์แฝงขนาด $512$ มิติเป็นเวกเตอร์ย่อยขนาด $64 \times 128 = 8,192$ มิติ
2. ใช้การทำ Transposed Convolution 1 มิติ (`ConvTranspose1D`) ร่วมกับ `GELU` เพื่อกระจายรายละเอียดและรูปร่างคลื่นกลับสู่ความละเอียดสูงขึ้น
3. ประมวลผลผ่าน Linear Adjustment layer เพื่อได้สัญญาณขนาด $1 \times 4000$ บนพิกัด ppm ที่แม่นยำ

### 5.2 สถาปัตยกรรมและทฤษฎี Energy-Based Models (EBM)
**Energy-Based Model** (LeCun et al., 2006) เป็นการเรียนรู้แบบไม่อาศัยตัวนำความน่าจะเป็นโดยตรง แต่ใช้การแปลงระบบคณิตศาสตร์ให้อยู่ในรูปของการให้ค่าพลังงานสเกลาร์ (Scalar energy score $E(x)$) แก่ข้อมูลโครงสร้างนำเข้า $x$:
* ข้อมูลที่มีความถูกต้อง มีโครงสร้างทางชีวเคมีที่เป็นไปตามกฎฟิสิกส์ จะได้ค่าพลังงานต่ำ ($E(x) \to 0$)
* ข้อมูลที่มีสิ่งผิดปกติ ปนเปื้อน หรือเกิดความขัดแย้งเชิงทฤษฎีกลศาสตร์ควอนตัม จะได้ค่าพลังงานสูงมาก ($E(x) \gg 0$)

```
                  Low Energy State                  High Energy State
                    (Clean NMR)                      (Anomaly/Ghost)
                    
                     _/\_/\_                            _/\_/\_
                    /       \                          /   |   \  <-- Spurious peak
                   /         \                        /    |    \
             ─────────────────────                ───────────────────
                  Energy ~ 0.4                       Energy ~ 1.5
```

### 5.3 การซอยย่อยเขตเคมี (Segmented Chemical Domains) ใน LocalizedPatchEBM
เพื่อให้การตรวจสอบกฎฟิสิกส์เคมีอธิบายได้ในระดับพิกัดที่แท้จริง `LocalizedPatchEBM` ได้รับการออกแบบให้ทำการแบ่งสเปกตรัมที่ถูกจำลองใหม่ออกเป็น 3 พื้นที่ปฏิบัติการหลักสอดคล้องกับพฤติกรรมโครงสร้างทางเคมีของสารในธรรมชาติ:
1. **Aliphatic Zone ($0.5 - 3.0\text{ ppm}$):** โซนของหมู่ไฮโดรคาร์บอนโซ่ตรง (เช่น อะมิโนโปรตีนและกรดไขมันอิ่มตัว) พีคโซนนี้มีลักษณะเป็นพีคแหลมสูงจากโครงสร้างพันธะอะลิฟาติกเดี่ยว
2. **Carbohydrate/Sugar Zone ($3.0 - 5.5\text{ ppm}$):** โซนของสารกลุ่มน้ำตาลและคาร์โบไฮเดรต มีความซับซ้อนของโครงสร้าง multiplet สูงมาก และมีแนวโน้มการทับซ้อนของสัญญาณสูง
3. **Aromatic Zone ($5.5 - 9.0\text{ ppm}$):** โซนของกลุ่มสารประกอบอะโรมาติกวงแหวนเบนซีนและฟีนอลิก พีคโซนนี้มักแยกตัวห่างกันและมีความเข้มข้นต่ำแต่มีความสำคัญทางคลินิกมาก

คณิตศาสตร์ของแบบจำลองคำนวณพลังงานรวมแบบถ่วงน้ำหนัก (Weighted global energy calculation):
$$E_{\text{global}} = w_{\text{aliphatic}} E(\text{patch}_1) + w_{\text{sugar}} E(\text{patch}_2) + w_{\text{aromatic}} E(\text{patch}_3)$$
โดยที่กำหนดสัดส่วนความสำคัญเชิงคลินิกสำหรับสิ่งส่งตรวจพืชเป็น $0.4 \times E_1 + 0.4 \times E_2 + 0.2 \times E_3$ ตามลำดับ

เมื่อค่า $E_{\text{global}}$ เกินเกณฑ์แจ้งเตือน (EBM Threshold เช่น $1.1$ หรือ $1.2$) โมเดลจะตรวจพบทันทีว่ามี **Ghost Peak** ปรากฏตัว และทำการสลักขั้วลบเพื่อลดความเข้มของสัญญาณแปลกปลอมลง (Suppression) พร้อมบันทึกพิกัด ppm ไปที่ระบบ telemetry

## Chapter 6: Stage 4 — Constrained Dynamic Time Warping (DTW) & Hybrid Bipartite Matching

### 6.1 ข้อจำกัดของการใช้ระยะทางแบบยูคลิดดิก (Euclidean Distance)
การเปรียบเทียบความคล้ายคลึงของสัญญาณ NMR โดยใช้การวัดระยะยูคลิดตรงๆ (Euclidean Distance) หรือ Pearson Correlation ระหว่างเวกเตอร์สองตัว มีจุดบกพร่องทางคณิตศาสตร์อย่างร้ายแรง:
* หากพีคเดียวกันเลื่อนพิกัดออกไปด้านข้างเพียง $0.02\text{ ppm}$ เวกเตอร์ทั้งสองจะจับคู่สัญญาณกันแบบจุดต่อจุดไม่เจอกันเลย ส่งผลให้ค่า Euclidean Distance สูงมาก และโมเดลจะรายงานว่า "ข้อมูลไม่ตรงกันเลย" (False Negative)

### 6.2 ทฤษฎีและอัลกอริทึม Dynamic Time Warping (DTW)
**Dynamic Time Warping (DTW)** เป็นอัลกอริทึมค้นหาแนวการจับคู่ที่ดีที่สุดระหว่างอนุกรมเวลาที่มีการเปลี่ยนแปลงความเร็วหรือพิกัดเวลาที่ไม่เท่ากัน (Non-linear alignment mapping) โดยใช้ Dynamic Programming

ให้สเปกตรัมที่ถอดรหัสแล้วเป็น $A = [a_1, a_2, ..., a_n]$ และสเปกตรัมอ้างอิงเป็น $B = [b_1, b_2, ..., b_m]$ ระบบจะสร้างตารางระยะทางสะสม $D$ ขนาด $n \times m$ โดยที่:
$$D(i, j) = \text{cost}(a_i, b_j) + \min \left( D(i-1, j), D(i, j-1), D(i-1, j-1) \right)$$

* **การบังคับใช้ Sakoe-Chiba Constraint:**
  เพื่อจำกัดไม่ให้สมการคำนวณหาเส้นทางเบี่ยงเบนออกนอกความเป็นจริงเชิงฟิสิกส์ (เช่น ยืดพีค Lactate ไปจับคู่กับ Sucrose ที่อยู่ห่างไกล) จึงมีการจำกัดขอบเขตการค้นหาเส้นทางบิดเบี้ยวด้วยรัศมี **Sakoe-Chiba Band (radius=10)** เพื่อรับประกันว่าพีคจะได้รับอนุญาตให้ขยับเปรียบเทียบในกรอบพิกัดที่สมเหตุสมผลเท่านั้น ช่วยลดการใช้หน่วยความจำและความเร็วการประมวลผลได้อย่างก้าวกระโดด

```
              Unconstrained DTW                   Sakoe-Chiba Constrained (radius=10)
              
                 j                                    j
                 ▲                                    ▲
                 │   /                                │    /
                 │  /                                 │  / \   <-- Restricted search
                 │ /                                  │ /   \      band
                 └────────► i                         └────────► i
```

* **การแก้ไขปัญหาระยะทางเวกเตอร์ Euclidean ใน Scipy:**
  > [!IMPORTANT]
  > ในขั้นตอนการพัฒนาระบบ NMR ดั้งเดิม พบปัญหาระบบค้างและบั๊ก `ValueError: Input vector should be 1-D` จากฟังก์ชัน `scipy.spatial.distance.euclidean` เนื่องจากฟังก์ชันดังกล่าวพยายามรับอินพุต scalar ทั่วไปมาแปลงเวกเตอร์ความคล้าย
  > ทางวิศวกรได้พัฒนาและปรับปรุงเป็นฟังก์ชันความต่างสัมบูรณ์ประสิทธิภาพสูงในรูปแบบแลมบ์ดา:
  > `dist = lambda a, b: abs(a - b)`
  > ปลดล็อกคอขวด ทำให้ระบบ DTW สามารถรันได้เรียลไทม์ภายในเวลาเสี้ยววินาที

### 6.3 สูตรคำนวณคะแนนความเชื่อมั่นผสมแบบไฮบริด (Hybrid Score)
เพื่อให้ความแม่นยำในการระบุอัตลักษณ์สารมีคุณลักษณะที่แพทย์ทางคลินิกสามารถอธิบายและตรวจสอบที่มาที่ไปได้ ระบบจึงได้คิดค้น **สูตรคำนวณคะแนนไฮบริด (Hybrid Score)** ซึ่งเป็นการคำนวณถ่วงน้ำหนักข้อมูลเชิงลึก 3 มิติ:
$$\text{Match Confidence} = 0.45 \times \text{Peak Assignment} + 0.35 \times \text{DTW Similarity} + 0.20 \times \sigma(-E_{\text{ebm}})$$

โดยที่:
1. **Peak Assignment Score (น้ำหนัก 45%):** คะแนนการจับคู่ยอดพีครายตัวแบบ Bipartite Matching ในกรอบความเบี่ยงเบนจำกัด ±0.03 ppm
2. **DTW Similarity (น้ำหนัก 35%):** ความคล้ายคลึงของโครงสร้างรูปร่างเชิงพิกัดหลังจัดแนวแล้ว คำนวณเป็นค่าผกผันของระยะทาง: $\frac{1}{1 + D_{\text{dtw}}}$
3. **EBM Physics Prior (น้ำหนัก 20%):** คะแนนจำกัดสิทธิความเป็นไปได้เชิงธรรมชาติโมเลกุล โดยผ่านฟังก์ชันเปิดขอบเขต Sigmoid: $\sigma(-E_{\text{ebm}})$

การรวมสามมิตินี้ช่วยให้แน่ใจว่าสารที่ได้คะแนนสูงต้องมีคุณสมบัติครบ 3 มิติ คือ มียอดพีคตรง, รูปร่างสเปกตรัมสอดคล้อง, และไม่มีความผิดปกติของโครงสร้างฟิสิกส์เคมี

## Chapter 7: สถาปัตยกรรม Full-Stack & วงจรสื่อสาร Active Learning Telemetry

### 7.1 ผังปฏิสัมพันธ์ระหว่าง FastAPI Backend และ Astro Frontend
สถาปัตยกรรมของแพลตฟอร์มนี้แบ่งแยกการทำงานออกอย่างอิสระ (Decoupled Microservice Architecture):

```
       [ Astro Frontend: Port 4321 ]
        │  ▲
        │  │  1. Request simulated sample via GET /api/sample
        ▼  │  2. Receive raw, ppm axis & ground truth annotations
     [ FastAPI Backend Server: Port 8000 ]
        │  ▲
        │  │  3. Submit spectrum for AI inference via POST /api/analyze
        ▼  │  4. Run PyTorch inference pipeline (1D-CNN, ODE, EBM, DTW)
     [ Deep Learning Core Models ]
        │
        ▼ (If Ghost Peak Detected)

### 7.2 วงจรอัตโนมัติ Active Learning และการบันทึก telemetry
เมื่อระบบ EBM Physics Verifier (Stage 3) ตรวจพบพลังงานความไม่ถูกต้องสูงเกินค่าวิกฤตที่ยอมรับได้ ระบบจะดำเนินการช่วยเหลือทางเทคนิคทันที:
1. **Real-time UI alert:** แดชบอร์ด Astro จะแจ้งผลลัพธ์เป็นกล่องเตือนสีส้มแอมเบอร์ แสดงสถานะ `ANOMALY_CLEARED` ซึ่งบ่งบอกว่าตรวจพบความผิดปกติแต่ตัวประมวลผล EBM ได้สกัดออกให้แล้ว
2. **Background Callback Execution:** ฝั่ง Frontend ดำเนินการยิงข้อมูลผ่านกระบวนการ AJAX ร้องขอเชิงรุกเงียบๆ (Silent background POST) ไปที่เซิร์ฟเวอร์ในเส้นทาง `/api/telemetry`
3. **Telemetry Trace Log Appending:** เซิร์ฟเวอร์บันทึกข้อมูลร่องรอยการตรวจจับความผิดปกติลงสู่ไฟล์ `anomalous_peaks_feedback.json` โดยอัตโนมัติ:

```json
[
    {
        "sample_id": "NMR-SAMPLE-2026-X99",
        "timestamp": "2026-05-28T19:46:12.304Z",
        "raw_energy_score": 1.542,
        "energy_threshold": 1.1,
        "anomalous_peak_ppm": 4.15,
        "details": "High-intensity single peak violating J-coupling ratios in carbohydrate zone.",
        "action": "suppressed_and_flagged"
    }
]
```

ประวัติความผิดปกติเหล่านี้ทำหน้าที่เป็นชุดข้อมูลฝึกสอนเพิ่มเติม (Active learning corpus) เพื่อนำมารีเทรนและปรับแต่งความละเอียดในการคัดกรองสัญญาณในอนาคตได้อย่างยั่งยืน
