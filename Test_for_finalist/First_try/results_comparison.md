## 🔬 สรุปรายผลการทดลอง

### 1. EXP01: Cosine Similarity Baseline
*   **สถานะ:** ✅ เสร็จสมบูรณ์
*   **ผลการทดสอบ:** ด้วยข้อมูล Mock 1,000 ตัวอย่าง 38 สารเคมี
    *   **Best Threshold:** 0.0400 (ใช้เพื่อตัดสินใจว่ามีสารนี้หรือไม่)
    *   **ข้อสังเกต:** โมเดลมีความสามารถในการระบุตัวตน (Recall) สูงมาก (~95%) ซึ่งหมายความว่ามันมักจะไม่พลาดสารที่มีอยู่จริง แต่ Precision อยู่ที่ระดับ ~88.6% ซึ่งแปลว่ายังมีการทำนายผิด (False Positive) อยู่บ้าง โดยเกิดจากการซ้อนทับของพีค (Overlapping Peaks)
    *   **ไฟล์รายงานฉบับเต็ม:** `EXP01_cosine_baseline/results/exp01_report.txt`

### 2. EXP05: NMF / ICA Decomposition
*   **สถานะ:** ✅ เสร็จสมบูรณ์
*   **ผลการทดสอบ:** ด้วยข้อมูล Mock 1,000 ตัวอย่าง 38 สารเคมี (Unsupervised)
    *   **Best Relative Threshold:** 0.012165
    *   **ข้อสังเกต:** โมเดลมีความสามารถในการทำงานได้แม่นยำ (Precision) สูงขึ้นเล็กน้อย (~90.8%) เมื่อเทียบกับแบบรวบยอด แต่จุดอ่อนที่ชัดเจนคือ Recall ลดลงไปมาก (เหลือ ~62.6%) 
    *   **ทำไม Recall ต่ำ?:** เนื่องจากพีคมีการซ้อนทับกัน (Overlapping) มาก เมื่อ NMF พยายามแยกส่วนประกอบออกมาเป็น 38 Components อย่างอิสระ บาง Components จึงเป็นการรวมพีคจากหลายสาร (Fragment) ทำให้รูปร่างไม่เหมือนสารเคมีอ้างอิงเป๊ะๆ (Average Cosine Sim ของ Component เทียบกับสารจริงอยู่ที่ 0.7176) ทำให้ไม่สามารถแมปสารได้ครบถ้วน
    *   **ไฟล์รายงานฉบับเต็ม:** `EXP05_nmf_decomposition/results/exp05_report.txt`

### 3. EXP02: CNN + GRU (NMRQNet)
*   **สถานะ:** ✅ เสร็จสมบูรณ์ (อัปเดตเป็น 50 Epochs)
*   **ผลการทดสอบ:** ด้วยข้อมูล Mock แบ่งเป็น Train 800, Test 200 ตัวอย่าง 38 สารเคมี
    *   **ข้อสังเกต:** หลังจากเพิ่มเป็น 50 Epochs พบว่าค่า Loss เริ่มขยับลดลงอย่างเห็นได้ชัดในช่วง **Epoch ที่ 45-50** (ลดจาก 0.66 เหลือ 0.62) ทำให้โมเดลเริ่มหลุดจากอาการ "เดา 1 ทุกตัว" (Recall ลดลงจาก 100% เหลือ 94.7% และ Precision ขยับขึ้นมาเป็น 63.39%)
    *   **สรุปผล:** แม้โมเดลจะเริ่มเรียนรู้ได้ แต่โครงสร้างที่ลึกมาก (Conv 5 ชั้น + GRU) ทำให้มันเรียนรู้ได้ **ช้ามากๆ** และ 800 Samples ถือว่าน้อยไปสำหรับสถาปัตยกรรมนี้ ทำให้ได้ผลรวม (F1 = 75.5%) ต่ำกว่า Rule-Based อย่าง Cosine Baseline (91.1%) เสียอีก
    *   **ไฟล์รายงานฉบับเต็ม:** `EXP02_cnn_nmrqnet/results/exp02_report.txt`

### 4. EXP03: 1D-ResNet
*   **สถานะ:** ✅ เสร็จสมบูรณ์
*   **ผลการทดสอบ:** ด้วยข้อมูล Mock แบ่งเป็น Train 800, Test 200 ตัวอย่าง 38 สารเคมี (Supervised / 50 Epochs)
    *   **ข้อสังเกต:** โครงสร้าง Residual Block (Skip Connection) ทำงานได้ผลจริง! สังเกตจากค่า Loss ที่ลดลงเร็วและต่อเนื่องกว่า EXP02 มาก (จบที่ Loss 0.43 เทียบกับ EXP02 ที่จบ 0.62) และใช้เวลาเทรนเร็วกว่า 
    *   **สรุปผล:** ถึงแม้ Loss ตอนเทรนจะลดลงได้ดี แต่ผลลัพธ์บนฝั่ง Test Set กลับได้ F1-Score ที่ 72.6% ซึ่งพอๆ กับ EXP02 สาเหตุหลักมาจากโมเดลมีขนาดใหญ่เกินไปสำหรับข้อมูลเพียง 800 ตัวอย่าง (Overparameterization) ทำให้โมเดลเกิดอาการ Overfit คือจำข้อสอบแม่นแต่ทำข้อสอบใหม่ไม่ได้ และยังแก้ปัญหา Overlapping Peaks ได้ไม่เด็ดขาด
    *   **ไฟล์รายงานฉบับเต็ม:** `EXP03_resnet1d/results/exp03_report.txt`

### 5. EXP04: CNN + Transformer
*   **สถานะ:** ✅ เสร็จสมบูรณ์
*   **ผลการทดสอบ:** ด้วยข้อมูล Mock แบ่งเป็น Train 800, Test 200 ตัวอย่าง 38 สารเคมี (Supervised / 50 Epochs)
    *   **ข้อสังเกต:** โมเดลลูกผสมที่ใช้ CNN ในการสกัดและย่อความยาวคลื่นลง แล้วส่งเข้า Transformer Encoder ให้กลไก Attention จัดการจับคู่ความสัมพันธ์ของพีค สามารถทำคะแนนได้อย่างก้าวกระโดด! Loss ลดลงต่ำสุดที่ 0.30 
    *   **สรุปผล:** ได้คะแนนสมดุลทั้ง Precision และ Recall (~86%) รวมเป็น F1-Score: 85.92% ซึ่ง **สูงที่สุดในกลุ่มโมเดล Deep Learning ทั้งหมดที่เราทดลอง** ทิ้งห่าง EXP02 และ EXP03 ขาดลอย และใช้เวลาเทรนน้อยมาก (ราวๆ 43 วินาที) เนื่องจากมีการย่อความยาวของ Sequence ลงก่อนเข้า Transformer 
    *   **อย่างไรก็ตาม:** แม้จะทำได้ดีเยี่ยมสำหรับ Deep Learning แต่อาจจะยังเอาชนะวิธีพื้นฐานอย่าง Cosine Similarity (91.1%) ที่เปรียบเทียบจาก Rule-base ตรงๆ ไม่ได้ในแง่ของความแม่นยำรวม หากจะให้ชนะอาจจะต้องใช้ข้อมูล Train ระดับ 10,000+ ตัวอย่าง
    *   **ไฟล์รายงานฉบับเต็ม:** `EXP04_transformer/results/exp04_report.txt`

## 🏆 ข้อสรุปจากภาพรวม (Conclusion)
จากการทดลองเปรียบเทียบโมเดล 5 รูปแบบ ในสภาพแวดล้อมข้อมูลสังเคราะห์ (Mock Data: 1,000 ตัวอย่าง / 38 สารเป้าหมาย / คลื่นความยาว 10,001 จุด) พบว่า:

1. **Rule-Based (Cosine Baseline) เป็นผู้ชนะในแง่ของคะแนนรวม (F1 = 91.1%)** และทำงานได้รวดเร็วมาก เหมาะสำหรับการใช้งานจริงที่สุดในปัจจุบันหากมีข้อมูลอ้างอิง (Reference Library) ที่ชัดเจน 
2. **Deep Learning (Supervised)** 
   - **Transformer** (EXP04) เป็นสถาปัตยกรรมที่ดีที่สุดในฝั่ง AI (F1 = 85.9%) พิสูจน์ให้เห็นว่ากลไก Attention ช่วยให้โมเดลแยกแยะพีคที่ซ้อนทับกันได้ดีกว่า 
   - **CNN+GRU / ResNet** (EXP02, EXP03) สอบตกในแง่ของ F1-Score เนื่องจากโมเดลเกิด Overfitting กับข้อมูลที่มีจำนวนน้อย (800 ตัวอย่าง) ทำให้ทำนายแบบเอนเอียง (Bias toward positive class)
3. **Unsupervised (NMF/ICA)** (EXP05) มีข้อจำกัดเมื่อเจอข้อมูลที่มีความซับซ้อนสูงและพีคทับซ้อนกันมาก ทำให้ F1-Score ต่ำสุด (68.1%) ไม่เหมาะกับงานที่มีจำนวนสารผสมเยอะๆ (38 สาร) ในหนึ่งสเปกตรัม

**คำแนะนำสำหรับการพัฒนาต่อ (Next Steps):**
- นำวิธี **Cosine Similarity (EXP01)** ไปใช้เป็นโซลูชันหลัก (Production) สำหรับเฟสแรกของการพัฒนา
- หากต้องการปั้น AI ให้แม่นยำขึ้น สถาปัตยกรรม **Transformer (EXP04)** คือคำตอบที่ใช่ แต่ต้องสร้างข้อมูลฝึกสอนจำลองในระดับที่ใหญ่กว่านี้มาก (เช่น 10,000 - 100,000 ตัวอย่าง) เพื่อให้โมเดลทะลุกำแพงความแม่นยำของ Rule-Based ไปได้
